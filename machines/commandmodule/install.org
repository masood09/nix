#+TITLE: Installing NixOS on commandmodule
#+AUTHOR: Masood Ahmed
#+OPTIONS: toc:nil num:nil

This document describes the end-to-end process for installing NixOS on the
=commandmodule= laptop using:

- NixOS Live ISO (booted via USB)
- Encrypted ZFS root with passphrase
- Impermanence (ephemeral root, persistent /home)
- Flake-based configuration
- No desktop environment initially (TTY only, Niri planned)

* Overview

The installation flow:
- Boot the NixOS Live ISO via USB
- Connect to WiFi
- Partition and format the disk using =disko=
- Generate SSH + age keys
- Install NixOS using the flake
- Reboot into the installed system

* Prerequisites

- NixOS Live ISO on USB (built via =just build-iso=)
- WiFi credentials for initial setup
- GitHub repo with NixOS flake (=github:masood09/nix=)

* Boot NixOS Live ISO

Write the ISO to USB and boot the laptop from it. The live ISO already
contains your SSH public key as an authorized key.

* Connect to WiFi

The live ISO uses =wpa_supplicant=. Connect using =wpa_cli=:

#+begin_src bash
wpa_cli
#+end_src

Inside the =wpa_cli= interactive shell:

#+begin_src
> scan
> scan_results
> add_network
> set_network 0 ssid "<SSID>"
> set_network 0 psk "<PASSWORD>"
> enable_network 0
> quit
#+end_src

Verify connectivity:

#+begin_src bash
ping -c 3 github.com
#+end_src

* SSH into the Live Environment (Optional)

From another machine on the same network:

#+begin_src bash
ssh nixos@<LAPTOP_IP>
#+end_src

Switch to root:

#+begin_src bash
sudo su - root
#+end_src

* Identify the Disk

Find the disk ID for the laptop's NVMe/SSD:

#+begin_src bash
ls -la /dev/disk/by-id/ | grep nvme
#+end_src

Note the disk ID (e.g., =nvme-Samsung_SSD_...=). This must match the ID
configured in =machines/commandmodule/default.nix=.

* Partition and Format Disk with Disko

Run Disko using the flake configuration for =commandmodule=:

#+begin_src bash
nix --experimental-features "nix-command flakes" \
  --option tarball-ttl 0 \
  run --refresh github:nix-community/disko -- \
  --mode destroy,format,mount \
  --flake "github:masood09/nix#commandmodule"
#+end_src

During ZFS pool creation, you will be prompted for the encryption passphrase.
Choose a strong passphrase -- this will be required on every boot to unlock
the root pool.

At completion, all datasets should be mounted under =/mnt=.

Verify:

#+begin_src bash
mount | grep mnt
zfs list
#+end_src

* Prepare Persistent and Secret Directories

Create the directory structure required by impermanence and secret handling:

#+begin_src bash
mkdir -pv /mnt/nix/{secret/{initrd,age},persist/etc/ssh}
chmod 0700 /mnt/nix/secret
#+end_src

* 7. Generate SSH and age Keys

Generate the SSH host key used in initrd:

#+begin_src bash
ssh-keygen -t ed25519 -N "" -C "" \
  -f /mnt/nix/secret/initrd/ssh_host_ed25519_key
#+end_src

Generate an SSH keypair used for age encryption:

#+begin_src bash
ssh-keygen -t ed25519 -N "" -C "" \
  -f /mnt/nix/secret/age/ssh_ed25519_key
#+end_src

Convert the SSH public key to age format:

#+begin_src bash
nix-shell --extra-experimental-features flakes \
  -p ssh-to-age \
  --run 'cat /mnt/nix/secret/age/ssh_ed25519_key.pub | ssh-to-age'
#+end_src

Save the resulting age public key -- it must be added to =.sops.yaml= in
the flake repository and secrets re-encrypted before the first deploy.

* Generate hardware-configuration.nix

Generate the hardware configuration for the laptop:

#+begin_src bash
nixos-generate-config --root /mnt --no-filesystems --show-hardware-config \
  > /tmp/hardware-configuration.nix
#+end_src

Copy this file into the flake repository at
=machines/commandmodule/hardware-configuration.nix= and commit it before
proceeding with the install.

* Install NixOS

Install NixOS using the flake configuration for =commandmodule=:

#+begin_src bash
nixos-install \
  --no-root-passwd \
  --root /mnt \
  --flake github:masood09/nix#commandmodule \
  --option tarball-ttl 0
#+end_src

* Cleanup and Reboot

Unmount all filesystems, export ZFS pools, and reboot:

#+begin_src bash
umount -Rl /mnt
zpool export -a
reboot
#+end_src

Remove the USB drive during reboot.

* Post-Install Notes

After reboot:

- You will be prompted for the ZFS encryption passphrase during boot.
- Verify ZFS pools:
  #+begin_src bash
  zpool status
  zfs list
  #+end_src
- Confirm =/= is ephemeral (rolled back on each boot).
- Confirm =/home= persists across reboots.
- Connect to WiFi via =nmcli=.
- The age public key from step 7 must be added to =.sops.yaml= and secrets
  re-encrypted before the first deploy via =just deploy=.
