#+TITLE: Installing NixOS on trialunit
#+AUTHOR: Masood Ahmed
#+OPTIONS: toc:nil num:nil

This document describes the end-to-end process for installing NixOS on the
=trialunit= Proxmox VM using:

- NixOS Live ISO (booted via Proxmox virtual media)
- ZFS root with Disko
- Encrypted ZFS pools (rpool, dpool)
- Flake-based configuration
- Impermanence + secrets management

* Overview

The installation flow:
- Boot the NixOS Live ISO via Proxmox console
- Partition and format disks using =disko=
- Generate ZFS encryption keys
- Generate SSH + age keys
- Install NixOS using the flake
- Reboot into the installed system

* Prerequisites

- Proxmox access with the NixOS ISO uploaded
- GitHub repo with NixOS flake (=github:masood09/nix=)

* 1. Boot NixOS Live ISO via Proxmox

1. Upload the NixOS ISO to Proxmox storage (built via =just build-iso=)
2. Attach the ISO to the trialunit VM as a CD/DVD drive
3. Boot the VM from the ISO

Once booted into the NixOS live environment, enable SSH access:

#+begin_src bash
mkdir -p ~/.ssh
curl "https://github.com/masood09.keys" > ~/.ssh/authorized_keys
#+end_src

From your local machine, connect and switch to root:

#+begin_src bash
ssh nixos@IP_ADDRESS
#+end_src

#+begin_src bash
sudo su - root
#+end_src

* 2. Create ZFS Encryption Key Material

Create the directory for ZFS encryption keys and generate a strong passphrase.
This key will later be persisted into the installed system.

#+begin_src bash
install -d -m 0700 /nix/secret
tr -dc 'A-Za-z0-9_@%+=:,.~-' </dev/urandom | head -c 64 > /nix/secret/dpool_tank_key
chmod 0400 /nix/secret/dpool_tank_key
#+end_src

Verify permissions:

#+begin_src bash
ls -l /nix/secret/dpool_tank_key
#+end_src

Expected output:

#+begin_example
-r-------- 1 root root 64 dpool_tank_key
#+end_example

Verify key length (do not print contents):

#+begin_src bash
wc -c /nix/secret/dpool_tank_key
#+end_src

* 3. Partition and Format Disks with Disko

Run Disko using the flake configuration for =trialunit=. This will:

- Destroy existing partition tables
- Create GPT layouts
- Create and mount ZFS pools and datasets

#+begin_src bash
nix --experimental-features "nix-command flakes" \
  --option tarball-ttl 0 \
  run --refresh github:nix-community/disko -- \
  --mode destroy,format,mount \
  --flake "github:masood09/nix#trialunit"
#+end_src

At completion, all datasets should be mounted under =/mnt=.

* 4. Prepare Persistent and Secret Directories

Create the directory structure required by impermanence and secret handling
inside the mounted system:

#+begin_src bash
mkdir -pv /mnt/nix/{secret/{initrd,age},persist/etc/ssh}
chmod 0700 /mnt/nix/secret
#+end_src

Copy the previously generated ZFS encryption key into the persistent system:

#+begin_src bash
cp /nix/secret/dpool_tank_key /mnt/nix/secret/
#+end_src

* 5. Generate SSH and age Keys

Generate the SSH host key used in initrd:

#+begin_src bash
ssh-keygen -t ed25519 -N "" -C "" \
  -f /mnt/nix/secret/initrd/ssh_host_ed25519_key
#+end_src

Generate an SSH keypair used for age encryption:

#+begin_src bash
ssh-keygen -t ed25519 -N "" -C "" \
  -f /mnt/nix/secret/age/ssh_ed25519_key
#+end_src

Convert the SSH public key to age format:

#+begin_src bash
nix-shell --extra-experimental-features flakes \
  -p ssh-to-age \
  --run 'cat /mnt/nix/secret/age/ssh_ed25519_key.pub | ssh-to-age'
#+end_src

Save the resulting age public key -- it must be added to =.sops.yaml= in
the flake repository.

* 6. Install NixOS

Install NixOS using the flake configuration for =trialunit=:

#+begin_src bash
nixos-install \
  --no-root-passwd \
  --root /mnt \
  --flake github:masood09/nix#trialunit \
  --option tarball-ttl 0
#+end_src

* 7. Cleanup and Reboot

Unmount all filesystems, export ZFS pools, and reboot into the installed system:

#+begin_src bash
umount -Rl /mnt
zpool export -a
reboot
#+end_src

* Post-Install Notes

After reboot:

- Verify ZFS pools:
  #+begin_src bash
  zpool status
  zfs list
  #+end_src

- Confirm encrypted pools unlocked correctly.
- Verify networking, SSH access, and persistence.
- The age public key from step 5 must be added to =.sops.yaml= and secrets
  re-encrypted before the first deploy.
