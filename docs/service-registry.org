* Purpose
This document is the *single source of truth* for homelab services, users, ports, and IDs.

Goals:
 * Avoid port collisions
 * Reuse consistent user/group IDs
 * Make adding new services trivial and predictable
 * Serve as design + operational documentation

This file should be consulted *before* adding any new service, container, or systemd unit.

* Conventions

** UID/GID Allocation
- Human users: 1000+
- Service users: 3000+ (UID always equals GID)

** Port Allocation
- Infrastructure (DNS, SSH): well-known or low ports
- Application services: 89xx block for homelab services
- Upstream defaults (Grafana 3000, Prometheus 9090, etc.) are kept as-is
- Metrics endpoints use upstream conventions (9xxx range)
