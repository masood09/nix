* Overview

Multi-machine homelab infrastructure using Nix Flakes. Manages 7 NixOS servers, 3 NixOS desktops, and 2 macOS machines with home-manager, sops-nix secrets, ZFS, and impermanence.

* Service Module Pattern

Every service follows the same structure:

#+begin_example
modules/services/<name>/
  options.nix      homelab.services.<name>.* option declarations
  default.nix      Implementation gated by lib.mkIf <service>.enable
#+end_example

Each service module can optionally integrate with:

| Integration   | Option                                    | Effect                                           |
|---------------+-------------------------------------------+--------------------------------------------------|
| ZFS dataset   | =homelab.services.<name>.zfs.enable=        | Dedicated dataset created and mounted at boot    |
| Impermanence  | automatic                                 | Persist data dirs if ZFS is not used             |
| Restic backup | =homelab.zfs.datasets.<name>.restic.enable= | Dataset included in nightly restic backup        |
| Secrets       | =config.sops.secrets."<path>".path=         | Decrypted at runtime by sops-nix                 |
| Reverse proxy | =services.caddy.virtualHosts=               | Service registers its own Caddy vhost            |
| Permissions   | =lib/systemd-helpers.nix=                   | Oneshot service fixes dir ownership before start |

* homelab.* Option Namespace

All machine identity and service configuration flows through the =homelab.*= namespace:

Core cross-platform identity options (=role=, =purpose=, =primaryUser=, and =networking.hostName=) are declared once in =modules/shared/options.nix= and then consumed by the NixOS and Darwin module trees.

#+begin_example
homelab
  +-- role             "server" | "desktop"
  +-- purpose          Free-text machine description
  +-- primaryUser      { userName, userId }
  +-- networking       { hostName, domain }
  +-- impermanence     bool — ephemeral root enabled
  +-- isRootZFS        bool — root is on ZFS
  +-- stylix          { enable, scheme, polarity, wallpaper } (NixOS-level theming)
  +-- programs
  |     +-- stylix    { enable, scheme, polarity, wallpaper } (HM-level theming)
  |     +-- emacs.enable
  |     +-- ...
  +-- services
  |     +-- backup     { enable, serviceUnits, extraPaths, backupRoot }
  |     +-- restic     { enable, pruneOpts }
  |     +-- caddy      { enable }
  |     +-- immich     { enable, zfs.enable, dataDir, ... }
  |     +-- ...        (27 services total)
  +-- zfs
        +-- datasets   Attrset of declarative ZFS datasets
#+end_example

  headscale-dns.sops.yaml          Scoped (meshcontrol, trialunit)

  secrets.sops.yaml                Per-host (that machine + dev keys)
#+end_example

* Storage Architecture

Most machines use ZFS with encryption and ephemeral root:

#+begin_example
Boot:     /boot (mirrored ESP on multi-disk machines)
Root:     rpool/root (rolled back to blank snapshot on every boot)
Persist:  /nix/persist (bind-mounted via impermanence)
Services: rpool/root/var/lib/<service> (dedicated datasets per service)
Backup:   rpool/root/var/backup/postgresql (pg_dump target)
#+end_example

Non-ZFS machines (arrakis, caretaker) use LUKS-encrypted ext4 with tmpfs root:

#+begin_example
Boot:     /boot (EFI, vfat)
Root:     tmpfs (wiped on every reboot)
Nix:      /dev/mapper/cryptroot → /nix (ext4, persistent)
Persist:  /nix/persist (bind-mounted via impermanence)
#+end_example

See [[file:backup.org][backup.org]] for the nightly backup pipeline.

* Key Helpers

** =lib/persistence-helpers.nix= — =mkPersistenceDirs=

Generates impermanence bind-mount directory entries with a three-part guard: =impermanence && !isRootZFS && !zfsEnable=. Services that manage their own ZFS dataset bypass impermanence entirely.

** =lib/systemd-helpers.nix= — =mkPermissionService=

Generates a oneshot systemd service that fixes directory ownership and permissions before the main service starts. Integrates with ZFS dataset ordering so the directory exists before permissions are set.

** =lib/zfs-options.nix= — =mkZfsOptions=

Builder for consistent =homelab.services.<name>.zfs.*= option declarations. Provides =enable=, =dataset=, and =mountpoint= with sensible defaults derived from the service name.
