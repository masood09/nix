#+TITLE: Restoring Karakeep from Restic Backup
#+AUTHOR: Masood Ahmed
#+OPTIONS: toc:nil num:nil

This document describes a *safe, repeatable procedure* for restoring the
Karakeep data files from Restic backups staged via =/mnt/nightly_backup=.

The workflow:
- Verifies available backups
- Restores data to a temporary location
- Stops Karakeep service
- Restores data files with correct permissions
- Restarts service cleanly

* Backup Information

- Data path: =/var/lib/karakeep=

* 1. Verify Available Snapshots

Confirm the snapshot you intend to restore:

#+begin_src bash
sudo restic-backup snapshots
#+end_src

* 2. Restore the Latest Snapshot to a Temporary Location

Restore the Karakeep data into =/tmp=:

#+begin_src bash
sudo restic-backup restore latest \
  --include /mnt/nightly_backup/karakeep \
  --target /tmp
#+end_src

This will create the following directory structure:

#+begin_example
/tmp/mnt/nightly_backup/karakeep/
#+end_example

* 3. Stop Karakeep Service

Stop the Karakeep service before restoring data:

#+begin_src bash
sudo systemctl stop karakeep.service
#+end_src

* 4. (Recommended) Take Pre-Restore ZFS Snapshot

Take a ZFS snapshot for quick rollback if something goes wrong:

#+begin_src bash
sudo zfs snapshot -r rpool/encrypted/services/karakeep@pre-restore
#+end_src

* 5. Clear the Existing Data Directory

Remove all existing contents while preserving the directory itself:

#+begin_src bash
sudo find /var/lib/karakeep -mindepth 1 -delete
#+end_src

* 6. Restore Data Files

Copy the restored data into the live directory:

#+begin_src bash
sudo rsync -a --delete \
  /tmp/mnt/nightly_backup/karakeep/ \
  /var/lib/karakeep/
#+end_src

Fix ownership so the service can read and write all files:

#+begin_src bash
sudo chown -R karakeep:karakeep /var/lib/karakeep
#+end_src

* 7. Start Karakeep Service

Restart the Karakeep service:

#+begin_src bash
sudo systemctl start karakeep.service
#+end_src

Verify service status:

#+begin_src bash
sudo systemctl status karakeep.service
#+end_src

* 8. Clean Up Temporary Files

Remove the temporary restore files:

#+begin_src bash
sudo rm -rf /tmp/mnt/nightly_backup/karakeep
#+end_src

Once the service is verified healthy, remove the pre-restore snapshot:

#+begin_src bash
sudo zfs destroy -r rpool/encrypted/services/karakeep@pre-restore
#+end_src

* Notes & Best Practices

- Always stop the Karakeep service before restoring data.
- =rsync --delete= ensures the restored state exactly matches the backup.
- The pre-restore ZFS snapshot allows instant rollback if the restore fails.
