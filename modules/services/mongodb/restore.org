#+TITLE: Restoring MongoDB Data from Restic Backup (ZFS Dataset)
#+AUTHOR: Masood Ahmed
#+OPTIONS: toc:nil num:nil

This document restores MongoDB from Restic backups that were staged under:

- =/mnt/nightly_backup/mongodb=

Because Restic restores files to the same paths captured in the snapshot, it
cannot directly restore these into =/var/lib/mongodb/...= in one step.

Instead, we restore into a *ZFS staging dataset* (not =/tmp=), then copy into
place.

* Backup Information

- Service: =MongoDB=
- Live base path: =/var/lib/mongodb=

Stop command:

#+begin_src bash
sudo systemctl stop nightscout.service
sudo systemctl stop mongodb.service
#+end_src

Start command:

#+begin_src bash
sudo systemctl start mongodb.service
sudo systemctl start nightscout.service
#+end_src

* Verify Available Snapshots

#+begin_src bash
sudo restic-backup snapshots
#+end_src

Select a snapshot ID or use =latest=.

* Create a ZFS Staging Dataset (Avoid /tmp)

We create a staging dataset that will temporarily hold the restored
=/mnt/nightly_backup/...= tree. This keeps the extra restore space off of =/tmp=.

Choose a mountpoint, for example:

- =/mnt/restore-staging=

#+begin_src bash
# Create a staging dataset (adjust pool/dataset name as desired)
sudo zfs create -o mountpoint=/mnt/restore-staging dpool/tank/restore-staging

# Confirm it is mounted
mount | grep restore-staging || true
df -h /mnt/restore-staging
#+end_src


* Stop MongoDB and any services using MongoDB.

*IMPORTANT:* Clean Shutdown Required

MongoDB must be cleanly stopped before restoring.

#+begin_src bash
sudo systemctl stop nightscout.service
sudo systemctl stop mongodb.service
sudo systemctl status mongodb.service
#+end_src

Confirm no running mongod process:

#+begin_src bash
ps aux | grep mongod | grep -v grep || echo "MongoDB stopped"
#+end_src

* (Recommended) Take Pre-Restore ZFS Snapshot

This allows instant rollback.

Replace dataset if different in your config.

#+begin_src bash
sudo zfs snapshot -r fpool/fast/services/mongodb@pre-restore
#+end_src

Verify:

#+begin_src bash
zfs list -t snapshot | grep mongodb@pre-restore
#+end_src

* Clear Existing MongoDB Data

*WARNING:* Only do this after snapshotting.

#+begin_src bash
sudo find /var/lib/mongodb -mindepth 1 -delete
#+end_src

* Restore Snapshot into Staging Dataset

Because Restic restores to the original path, restoring into
=/mnt/restore-staging= will create:

=/mnt/restore-staging/mnt/nightly_backup/mongodb=

#+begin_src bash
SNAPSHOT=latest

sudo restic-backup restore "${SNAPSHOT}" \
--include /mnt/nightly_backup/mongodb \
--target /mnt/restore-staging
#+end_src

Expected layout:

#+begin_example
/mnt/mongodb-restore/mnt/nightly_backup/mongodb
#+end_example

* Copy Restored Data into Live Dataset

MongoDB requires correct ownership and permissions. Use rsync to preserve:
- Permissions
- Hardlinks
- xattrs
- ACLs

#+begin_src bash
sudo rsync -aAX --delete \
--chown=mongodb:mongodb \
/mnt/restore-staging/mnt/nightly_backup/mongodb/ \
/var/lib/mongodb/
#+end_src

Verify ownership:

#+begin_src bash
ls -ld /var/lib/mongodb
#+end_src

Should show:

#+begin_example
mongodb mongodb
#+end_example

* Start MongoDB

#+begin_src bash
sudo systemctl start mongodb.service
#+end_src

Verify:

#+begin_src bash
sudo systemctl status mongodb.service
#+end_src

Check logs:

#+begin_src bash
sudo journalctl -u mongodb -f
#+end_src

Validate Database Integrity

Connect locally:

#+begin_src bash
mongosh
#+end_src

Inside shell:

#+begin_src javascript
show dbs
use admin
db.runCommand({ connectionStatus: 1 })
#+end_src

If authentication is enabled (as in your config), verify root login works.

* Cleanup

Once satisfied:

*1. Destroy staging dataset*

#+begin_src bash
sudo zfs destroy -r dpool/tank/restore-staging
#+end_src

*2. Remove pre-restore snapshot*

#+begin_src bash
sudo zfs destroy -r fpool/fast/services/mongodb@pre-restore
#+end_src

Verify:

#+begin_src bash
zfs list -t snapshot | grep pre-restore || echo "Pre-restore snapshot removed"
#+end_src

* Rollback (If Something Goes Wrong)

If MongoDB fails to start or data is corrupt:

#+begin_src bash
sudo systemctl stop mongodb.service
sudo zfs rollback -r fpool/fast/services/mongodb@pre-restore
sudo systemctl start mongodb.service
#+end_src

* Start other services

Start any other services dependent on =MongoDB=

#+begin_src bash
sudo systemctl start nightscout.service
#+end_src
