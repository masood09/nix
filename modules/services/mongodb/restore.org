#+TITLE: Restoring MongoDB from Restic Backup
#+AUTHOR: Masood Ahmed
#+OPTIONS: toc:nil num:nil

This document restores MongoDB from Restic backups staged under
=/mnt/nightly_backup/mongodb=.

Because Restic restores files to the same absolute paths captured in the
snapshot, we restore into a *ZFS staging dataset* first, then copy into place.

* Backup Information

- Service: =MongoDB=
- Live base path: =/var/lib/mongodb=

* 1. Verify Available Snapshots

#+begin_src bash
sudo restic-backup snapshots
#+end_src

Select a snapshot ID or use =latest=.

* 2. Create a ZFS Staging Dataset

Create a staging dataset to temporarily hold the restored files:

#+begin_src bash
sudo zfs create -o mountpoint=/mnt/restore-staging dpool/tank/restore-staging

mount | grep restore-staging || true
df -h /mnt/restore-staging
#+end_src

* 3. Stop MongoDB and Dependent Services

*IMPORTANT:* MongoDB must be cleanly stopped before restoring.

#+begin_src bash
sudo systemctl stop nightscout.service
sudo systemctl stop mongodb.service
sudo systemctl status mongodb.service
#+end_src

Confirm no running mongod process:

#+begin_src bash
ps aux | grep mongod | grep -v grep || echo "MongoDB stopped"
#+end_src

* 4. (Recommended) Take Pre-Restore ZFS Snapshot

This allows instant rollback:

#+begin_src bash
sudo zfs snapshot -r fpool/fast/services/mongodb@pre-restore
#+end_src

Verify:

#+begin_src bash
zfs list -t snapshot | grep mongodb@pre-restore
#+end_src

* 5. Clear Existing MongoDB Data

#+begin_src bash
sudo find /var/lib/mongodb -mindepth 1 -delete
#+end_src

* 6. Restore Snapshot into Staging Dataset

#+begin_src bash
SNAPSHOT=latest

sudo restic-backup restore "${SNAPSHOT}" \
  --include /mnt/nightly_backup/mongodb \
  --target /mnt/restore-staging
#+end_src

Expected layout:

#+begin_example
/mnt/restore-staging/mnt/nightly_backup/mongodb/
#+end_example

* 7. Copy Restored Data into Live Dataset

Use =rsync -aAX= to preserve permissions, ACLs, and extended attributes:

#+begin_src bash
sudo rsync -aAX --delete \
  --chown=mongodb:mongodb \
  /mnt/restore-staging/mnt/nightly_backup/mongodb/ \
  /var/lib/mongodb/
#+end_src

Verify ownership:

#+begin_src bash
ls -ld /var/lib/mongodb
#+end_src

* 8. Start MongoDB

#+begin_src bash
sudo systemctl start mongodb.service
#+end_src

Verify:

#+begin_src bash
sudo systemctl status mongodb.service
sudo journalctl -u mongodb -f
#+end_src

* 9. Validate Database Integrity

Connect locally:

#+begin_src bash
mongosh
#+end_src

Inside shell:

#+begin_src javascript
show dbs
use admin
db.runCommand({ connectionStatus: 1 })
#+end_src

* 10. Start Dependent Services

Start any services that depend on MongoDB:

#+begin_src bash
sudo systemctl start nightscout.service
#+end_src

* 11. Clean Up

*1. Destroy the staging dataset:*

#+begin_src bash
sudo zfs destroy -r dpool/tank/restore-staging
#+end_src

*2. Remove the pre-restore snapshot:*

#+begin_src bash
sudo zfs destroy -r fpool/fast/services/mongodb@pre-restore
#+end_src

Verify:

#+begin_src bash
zfs list -t snapshot | grep pre-restore || echo "Pre-restore snapshot removed"
#+end_src

* Rollback (If Something Goes Wrong)

If MongoDB fails to start or data is corrupt:

#+begin_src bash
sudo systemctl stop mongodb.service
sudo zfs rollback -r fpool/fast/services/mongodb@pre-restore
sudo systemctl start mongodb.service
#+end_src

* Notes & Best Practices

- Always cleanly stop MongoDB before restoring to avoid journal corruption.
- =rsync -aAX= preserves extended attributes required by MongoDB.
- The pre-restore ZFS snapshot allows instant rollback if the restore fails.
