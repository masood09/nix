#+TITLE: Restoring Minecraft Server from Restic Backup
#+AUTHOR: Masood Ahmed
#+OPTIONS: toc:nil num:nil

This document describes a *safe, repeatable procedure* for restoring
Minecraft server world data from Restic backups staged via
=/mnt/nightly_backup=.

The workflow:
- Verifies available backups
- Restores world data to a temporary location
- Stops the target Minecraft server
- Takes a pre-restore ZFS snapshot
- Restores data files with correct permissions
- Restarts service cleanly

* Backup Information

- Primary service: =minecraft-server=
- Primary data path: =/var/lib/minecraft=
- Primary owner: =minecraft:minecraft=
- Primary ZFS dataset: =dpool/tank/services/minecraft=
- Second service: =minecraft2=
- Second data path: =/var/lib/minecraft2=
- Second owner: =minecraft2:minecraft2=
- Second ZFS dataset: =dpool/tank/services/minecraft2=
- Critical contents: world data (region files, player data, level.dat)
- Regenerated by Nix: =server.properties=, =whitelist.json=, =ops.json= (declarative mode overwrites on activation)

Use the matching service, data path, owner, and ZFS dataset for the server you are restoring.

* 1. Verify Available Snapshots

Confirm the snapshot you intend to restore:

#+begin_src bash
sudo restic-backup snapshots
#+end_src

* 2. Restore the Latest Snapshot to a Temporary Location

Restore the Minecraft data into =/tmp=:

#+begin_src bash
sudo restic-backup restore latest \
  --include /mnt/nightly_backup/minecraft \
  --target /tmp
#+end_src

This will create the following directory structure:

#+begin_example
/tmp/mnt/nightly_backup/minecraft/
#+end_example

* 3. Stop Minecraft Server

Stop the server before restoring data:

#+begin_src bash
sudo systemctl stop minecraft-server.service
#+end_src

For the second server, use:

#+begin_src bash
sudo systemctl stop minecraft2.service
#+end_src

* 4. (Recommended) Take Pre-Restore ZFS Snapshot

Take a ZFS snapshot for quick rollback if something goes wrong:

#+begin_src bash
sudo zfs snapshot dpool/tank/services/minecraft@pre-restore
#+end_src

For the second server, use:

#+begin_src bash
sudo zfs snapshot dpool/tank/services/minecraft2@pre-restore
#+end_src

* 5. Clear the Existing Data Directory

Remove all existing contents while preserving the directory itself:

#+begin_src bash
sudo find /var/lib/minecraft -mindepth 1 -delete
#+end_src

For the second server, use:

#+begin_src bash
sudo find /var/lib/minecraft2 -mindepth 1 -delete
#+end_src

* 6. Restore Data Files

Copy the restored data into the live directory:

#+begin_src bash
sudo rsync -a --delete \
  /tmp/mnt/nightly_backup/minecraft/ \
  /var/lib/minecraft/
#+end_src

For the second server, restore the matching backup path into =/var/lib/minecraft2=.

Fix ownership so the service can read and write all files:

#+begin_src bash
sudo chown -R minecraft:minecraft /var/lib/minecraft
#+end_src

For the second server, use:

#+begin_src bash
sudo chown -R minecraft2:minecraft2 /var/lib/minecraft2
#+end_src

* 7. Start Minecraft Server

Restart the server:

#+begin_src bash
sudo systemctl start minecraft-server.service
#+end_src

For the second server, use:

#+begin_src bash
sudo systemctl start minecraft2.service
#+end_src

Verify service status:

#+begin_src bash
sudo systemctl status minecraft-server.service
#+end_src

For the second server, use:

#+begin_src bash
sudo systemctl status minecraft2.service
#+end_src

Check the server log for successful world load:

#+begin_src bash
sudo journalctl -u minecraft-server.service -n 50 --no-pager
#+end_src

For the second server, use:

#+begin_src bash
sudo journalctl -u minecraft2.service -n 50 --no-pager
#+end_src

* 8. Clean Up Temporary Files

Remove the temporary restore files:

#+begin_src bash
sudo rm -rf /tmp/mnt/nightly_backup/minecraft
#+end_src

Once the service is verified healthy, remove the pre-restore snapshot:

#+begin_src bash
sudo zfs destroy dpool/tank/services/minecraft@pre-restore
#+end_src

For the second server, use:

#+begin_src bash
sudo zfs destroy dpool/tank/services/minecraft2@pre-restore
#+end_src

* Notes & Best Practices

- Always stop the Minecraft server before restoring data to avoid world corruption.
- The pre-restore ZFS snapshot allows instant rollback if the restore fails.
- After restore, verify the world loads correctly by checking journalctl logs for "Done" startup message.
- If restoring to a specific snapshot (not latest), replace =latest= with the snapshot ID from step 1.
- =server.properties=, =whitelist.json=, and =ops.json= do not need manual restore — Nix regenerates them from the module config.
