#+TITLE: Restoring Minecraft Server from Restic Backup
#+AUTHOR: Masood Ahmed
#+OPTIONS: toc:nil num:nil

This document describes a safe restore procedure for the heartbeat Minecraft
stack managed by =services.minecraft-servers= from =nix-minecraft=.

* Backup Information

- Data root: =/srv/minecraft=
- Owner: =minecraft:minecraft=
- ZFS dataset: =fpool/fast/services/minecraft=
- Proxy service: =minecraft-server-velocity.service=
- Awesome server service: =minecraft-server-awesome.service=
- Forever server service: =minecraft-server-forever.service=
- Awesome data path: =/srv/minecraft/awesome=
- Forever data path: =/srv/minecraft/forever=
- Regenerated by Nix: =server.properties=, =ops.json=, =eula.txt=, =velocity.toml=, and =forever/mods=

The old pre-refactor data paths are no longer used. If you need old world data,
copy it manually into the matching Awesome or Forever directory before starting
the server.

* 1. Verify Available Snapshots

Confirm the snapshot you intend to restore:

#+begin_src bash
sudo restic-backup snapshots
#+end_src

* 2. Restore the Latest Snapshot to a Temporary Location

Restore the Minecraft data into =/tmp=:

#+begin_src bash
sudo restic-backup restore latest \
  --include /mnt/nightly_backup/minecraft \
  --target /tmp
#+end_src

This creates:

#+begin_example
/tmp/mnt/nightly_backup/minecraft/
#+end_example

* 3. Stop Minecraft Services

Stop the proxy first, then the backend server you are restoring:

#+begin_src bash
sudo systemctl stop minecraft-server-velocity.service
sudo systemctl stop minecraft-server-awesome.service
#+end_src

For the Forever server, stop =minecraft-server-forever.service= instead.

* 4. Take a Pre-Restore ZFS Snapshot

Take a snapshot for quick rollback:

#+begin_src bash
sudo zfs snapshot fpool/fast/services/minecraft@pre-restore
#+end_src

* 5. Clear the Target Server Directory

Remove existing contents while preserving the server directory:

#+begin_src bash
sudo find /srv/minecraft/awesome -mindepth 1 -delete
#+end_src

For the Forever server, use =/srv/minecraft/forever=.

* 6. Restore Data Files

Copy restored data into the target server directory:

#+begin_src bash
sudo rsync -a --delete \
  /tmp/mnt/nightly_backup/minecraft/ \
  /srv/minecraft/awesome/
#+end_src

For the Forever server, restore into =/srv/minecraft/forever=.

Fix ownership:

#+begin_src bash
sudo chown -R minecraft:minecraft /srv/minecraft
#+end_src

* 7. Start Minecraft Services

Start the restored backend, then the proxy:

#+begin_src bash
sudo systemctl start minecraft-server-awesome.service
sudo systemctl start minecraft-server-velocity.service
#+end_src

For the Forever server, start =minecraft-server-forever.service=.

Verify status:

#+begin_src bash
sudo systemctl status minecraft-server-awesome.service
sudo systemctl status minecraft-server-velocity.service
#+end_src

Check logs:

#+begin_src bash
sudo journalctl -u minecraft-server-awesome.service -n 50 --no-pager
sudo journalctl -u minecraft-server-velocity.service -n 50 --no-pager
#+end_src

* 8. Clean Up

Remove temporary restore files:

#+begin_src bash
sudo rm -rf /tmp/mnt/nightly_backup/minecraft
#+end_src

Once the service is healthy, remove the pre-restore snapshot:

#+begin_src bash
sudo zfs destroy fpool/fast/services/minecraft@pre-restore
#+end_src

* Notes

- Always stop the target server before restoring data to avoid world corruption.
- Restore only world/server data that should persist; Nix regenerates declarative files on service start.
- The public entrypoint is Velocity on TCP 25565. Backend ports are bound to localhost only.
