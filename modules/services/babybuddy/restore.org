#+TITLE: Restoring Baby Buddy from Restic Backup
#+AUTHOR: Masood Ahmed
#+OPTIONS: toc:nil num:nil

This document describes a *safe, repeatable procedure* for restoring the
Baby Buddy PostgreSQL database and data files from Restic backups staged
via =/mnt/nightly_backup=.

The workflow:
- Verifies available backups
- Restores database dump and data to a temporary location
- Stops Baby Buddy service
- Drops and restores the PostgreSQL database
- Restores data files with correct permissions
- Restarts service cleanly

* Backup Information

- Database: =babybuddy=
- Database owner: =babybuddy=
- Database engine: PostgreSQL
- Data path: =/var/lib/babybuddy=

* 1. Verify Available Snapshots

Confirm the snapshot you intend to restore:

#+begin_src bash
sudo restic-backup snapshots
#+end_src

* 2. Restore the Latest Snapshot to a Temporary Location

Restore the PostgreSQL backup and Baby Buddy data into =/tmp=:

#+begin_src bash
sudo restic-backup restore latest \
  --include /mnt/nightly_backup/postgresql-backup/babybuddy.sql.gz \
  --target /tmp

sudo restic-backup restore latest \
  --include /mnt/nightly_backup/babybuddy \
  --target /tmp
#+end_src

This will create the following directory structure:

#+begin_example
/tmp/mnt/nightly_backup/postgresql-backup/babybuddy.sql.gz
/tmp/mnt/nightly_backup/babybuddy/
#+end_example

* 3. Stop Baby Buddy Service

Stop the Baby Buddy service before restoring the database:

#+begin_src bash
sudo systemctl stop podman-babybuddy.service
#+end_src

* 4. (Recommended) Take Pre-Restore ZFS Snapshot

Take a ZFS snapshot for quick rollback if something goes wrong:

#+begin_src bash
sudo zfs snapshot -r rpool/encrypted/services/babybuddy@pre-restore
sudo zfs snapshot -r dpool/tank/services/postgresql@pre-restore
#+end_src

* 5. Drop and Restore the Database

Remove the existing database before restoring from backup:

#+begin_src bash
sudo -u postgres psql -d postgres -c "DROP DATABASE babybuddy;"
sudo systemctl restart postgresql
#+end_src

Restore the database from the compressed SQL backup:

#+begin_src bash
sudo cp /tmp/mnt/nightly_backup/postgresql-backup/babybuddy.sql.gz /tmp/babybuddy.sql.gz
sudo chmod 644 /tmp/babybuddy.sql.gz

sudo -u babybuddy bash -lc \
  'gunzip -c /tmp/babybuddy.sql.gz | psql -v ON_ERROR_STOP=1 -d babybuddy'
#+end_src

* 6. Clear the Existing Data Directory

Remove all existing contents while preserving the directory itself:

#+begin_src bash
sudo find /var/lib/babybuddy -mindepth 1 -delete
#+end_src

* 7. Restore Data Files

Copy the restored data into the live directory:

#+begin_src bash
sudo rsync -a --delete \
  /tmp/mnt/nightly_backup/babybuddy/ \
  /var/lib/babybuddy/
#+end_src

Fix ownership so the service can read and write all files:

#+begin_src bash
sudo chown -R babybuddy:babybuddy /var/lib/babybuddy
#+end_src

* 8. Start Baby Buddy Service

Restart the Baby Buddy service:

#+begin_src bash
sudo systemctl start podman-babybuddy.service
#+end_src

Verify service status:

#+begin_src bash
sudo systemctl status podman-babybuddy.service
#+end_src

* 9. Clean Up Temporary Files

Remove the temporary restore files:

#+begin_src bash
sudo rm -f /tmp/babybuddy.sql.gz
sudo rm -rf /tmp/mnt/nightly_backup/babybuddy
sudo rm -f /tmp/mnt/nightly_backup/postgresql-backup/babybuddy.sql.gz
#+end_src

Once the service is verified healthy, remove the pre-restore snapshots:

#+begin_src bash
sudo zfs destroy -r rpool/encrypted/services/babybuddy@pre-restore
sudo zfs destroy -r dpool/tank/services/postgresql@pre-restore
#+end_src

* Notes & Best Practices

- Always stop Baby Buddy service before restoring the database.
- Use =ON_ERROR_STOP= to ensure restore failures abort immediately.
- The pre-restore ZFS snapshot allows instant rollback if the restore fails.
